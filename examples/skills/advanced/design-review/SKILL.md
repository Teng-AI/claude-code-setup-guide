# Design Review

Evaluate visual designs with a critical eye for aesthetics, usability, and effectiveness. Provide honest, actionable feedback that helps improve the design.

## When to Use
- User asks to review a design, screenshot, or webpage
- User wants feedback on how something looks
- User asks about UI/UX improvements
- User wants to know if a design is "good" or effective

## Review Process

### 1. First Impression (3-second test)
- What draws the eye first?
- Is the purpose/message immediately clear?
- Does it feel professional, trustworthy, modern?

### 2. Visual Hierarchy
- Is there a clear focal point?
- Do elements guide the eye in the right order?
- Are primary actions obvious and prominent?
- Is there appropriate contrast between important and secondary elements?

### 3. Typography
- Is text readable at intended sizes?
- Are there too many fonts? (Aim for 2-3 max)
- Is there consistent type scale and hierarchy?
- Is line height and letter spacing appropriate?
- Does body text have good line length (45-75 characters)?

### 4. Color
- Is the palette cohesive and intentional?
- Is there sufficient contrast for accessibility (WCAG AA: 4.5:1 for text)?
- Are colors used consistently for similar elements?
- Does the color scheme support the brand/mood?
- Is color used to guide attention appropriately?

### 5. Layout & Spacing
- Is there a consistent grid or alignment system?
- Is whitespace used effectively (not cramped or sparse)?
- Are related elements grouped together?
- Is spacing consistent (use a scale: 4, 8, 16, 24, 32, 48px)?
- Does the layout work at different screen sizes?

### 6. UI/UX Patterns
- Are interactive elements obviously clickable?
- Is navigation intuitive?
- Are form inputs clear with proper labels?
- Is feedback provided for user actions?
- Are common patterns followed (don't reinvent the wheel)?

### 7. Consistency
- Are similar elements styled the same way?
- Do buttons, links, and icons follow a consistent style?
- Is the visual language coherent throughout?

### 8. Polish & Details
- Are edges crisp and alignments precise?
- Are images high quality and properly sized?
- Are there any visual glitches or inconsistencies?
- Do micro-interactions feel smooth?

## Feedback Format

Structure feedback as:

1. **What's Working** - Acknowledge effective design choices
2. **Priority Issues** - 2-3 most impactful problems to fix
3. **Suggestions** - Specific, actionable improvements
4. **Quick Wins** - Small changes with big impact

## Principles to Apply

- **Less is more**: When in doubt, simplify
- **Alignment creates order**: Everything should align to something
- **Repetition creates unity**: Reuse styles consistently
- **Contrast creates hierarchy**: Make differences obvious
- **Proximity creates relationships**: Group related items
- **White space is not wasted space**: Let designs breathe

## Important Notes
- Be honest but constructive - the goal is improvement
- Prioritize feedback by impact (fix big issues first)
- Consider the context and audience
- Provide specific suggestions, not vague criticism
- Reference specific elements when giving feedback
- If reviewing a screenshot/image, use the Read tool to view it first
